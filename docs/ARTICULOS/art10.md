# ARTÍCULO 10 — FICHA TÉCNICA PARA CURSOR

## 1. Referencia

Valencia-Arias, A.; Méndez Velásquez, L. M.; Agudelo-Ceballos, E.; et al. (2025).  
“Educación virtual en el sector rural: análisis de la literatura científica”.  
Revista: Frontiers in Education.  
Volumen: 10.  
Artículo: 1567733.

## 2. Idea clave del artículo

El artículo analiza la educación virtual en el sector rural mediante una revisión sistemática y bibliométrica de la literatura científica.

La idea principal es que la educación virtual puede ayudar a reducir brechas educativas en zonas rurales, pero solo si se consideran las condiciones reales del contexto: conectividad limitada, falta de dispositivos, baja alfabetización digital, formación docente insuficiente, infraestructura tecnológica débil y necesidades sociales propias de las comunidades rurales.

El artículo deja claro que la virtualidad no debe verse como una solución automática. Para que funcione en zonas rurales, debe diseñarse de forma contextualizada, accesible y pedagógicamente adecuada.

## 3. Aporte específico al sistema

Este artículo aporta la base contextual rural para el sistema de nivelación académica adaptativa.

Su valor principal es que permite justificar que el sistema no debe diseñarse como una plataforma pensada solo para estudiantes urbanos o con buena conectividad. Debe considerar la realidad del VRAEM, donde pueden existir limitaciones de internet, dispositivos compartidos, baja alfabetización digital y desigualdad de acceso.

Este artículo ayuda a orientar decisiones sobre:

- Diseño simple y accesible.
- Contenidos livianos.
- Uso en contextos de baja conectividad.
- Apoyo docente.
- Inclusión digital.
- Seguimiento de problemas de acceso.
- Adaptación del sistema a comunidades rurales.

## 4. Decisión técnica derivada

A partir de este artículo, el sistema debe diseñarse con enfoque rural y de bajo consumo tecnológico.

Esto significa que el sistema debe priorizar:

- Interfaz sencilla.
- Bajo peso de recursos.
- Evitar depender de videos pesados como recurso principal.
- Registrar problemas de conectividad.
- Permitir que el docente identifique estudiantes con baja participación por problemas de acceso.
- Usar actividades cortas y descargables cuando sea posible.
- Diseñar funcionalidades comprensibles para docentes y estudiantes con diferentes niveles de alfabetización digital.

La decisión técnica principal es que el sistema debe adaptarse no solo al rendimiento académico, sino también a las condiciones tecnológicas del estudiante.

## 6. Técnicas que respalda

El artículo no se centra en un algoritmo de machine learning específico para estudiantes, sino en un análisis bibliométrico sobre educación virtual rural.

Para el sistema, respalda enfoques como:

- Analítica de participación.
- Monitoreo de acceso.
- Dashboards de brechas digitales.
- Reportes por zona o institución.
- Identificación de estudiantes con baja conectividad.
- Análisis de patrones de uso.
- Seguimiento docente basado en datos.
- Diseño de sistemas educativos contextualizados.
- Evaluación de inclusión digital.

Este artículo es más útil para diseñar el sistema desde el contexto rural que para elegir un algoritmo predictivo.

## 7. Regla o lógica aplicable al sistema

Este artículo permite derivar las siguientes reglas funcionales:

Si un estudiante no ingresa al sistema durante varios días, el sistema no debe asumir automáticamente desinterés o bajo rendimiento. Debe registrar posible problema de acceso.

Si varios estudiantes de una misma zona presentan baja participación, el sistema debe generar un reporte de posible brecha de conectividad.

Si un estudiante abandona actividades con frecuencia, el sistema debe permitir registrar si la causa fue dificultad académica, problema técnico o falta de conectividad.

Si el estudiante tiene baja conectividad, el sistema debe priorizar actividades cortas, livianas y de fácil carga.

Si el docente detecta baja participación, el sistema debe permitir registrar observaciones o intervención docente.

Si el sistema se usa en una institución rural, debe evitar recursos obligatorios que requieran alta velocidad de internet.

## 9. Resultado o evidencia relevante

El artículo identifica que la producción científica sobre educación virtual rural ha crecido de forma importante, especialmente después de la pandemia de COVID-19.

También muestra que los temas más relevantes en este campo incluyen:

- Inclusión digital.
- Conectividad.
- Formación docente.
- Aprendizaje móvil.
- Alfabetización digital.
- Salud mental.
- Educación online.
- Equidad educativa.
- Infraestructura tecnológica.

El artículo evidencia que la educación virtual rural tiene potencial para ampliar el acceso, pero también puede profundizar desigualdades si no se consideran las condiciones tecnológicas, sociales y pedagógicas de los estudiantes.

## 10. Limitación del artículo

Aunque el artículo es muy útil para contextualizar el sistema, presenta algunas limitaciones:

- Es un estudio bibliométrico, no una implementación de software.
- No propone una arquitectura técnica para una plataforma educativa.
- No desarrolla un modelo de machine learning.
- No evalúa directamente estudiantes de una zona rural específica como el VRAEM.
- Depende de Scopus y Web of Science, por lo que puede dejar fuera literatura local o informes regionales.
- Ofrece una visión panorámica, pero no profundiza en estrategias pedagógicas concretas.
- No define actividades específicas de nivelación académica.
- No entrega reglas adaptativas listas para implementar.

Por ello, debe usarse como base contextual y estratégica, no como guía técnica completa de implementación.

No se usará principalmente para definir algoritmos, sino para orientar decisiones de accesibilidad, conectividad, inclusión digital y acompañamiento docente.

Su aporte principal se aplicará en:

- Diseño simple de la interfaz.
- Registro de problemas de acceso.
- Reportes de participación.
- Alertas por inactividad.
- Adaptación a baja conectividad.
- Priorización de recursos livianos.
- Apoyo docente en estudiantes con acceso irregular.
- Interpretación contextual de los datos.

## 12. Aplicación directa al sistema VRAEM

En el sistema de nivelación académica adaptativa para estudiantes del VRAEM, este artículo permite establecer que la ruralidad no es un detalle secundario, sino una condición central del diseño.

La plataforma debe considerar que un estudiante puede tener bajo uso del sistema por causas externas, como:

- Internet limitado.
- Falta de dispositivo propio.
- Uso compartido de celular o computadora.
- Problemas familiares o económicos.
- Baja familiaridad con plataformas digitales.
- Falta de acompañamiento tecnológico.

Por eso, el sistema debe diferenciar entre:

- Bajo rendimiento académico.
- Baja participación por falta de acceso.
- Baja participación por dificultad con la plataforma.
- Riesgo académico real.
- Riesgo tecnológico o de conectividad.

## 13. Decisión técnica final derivada del Artículo 10

A partir de este artículo, el sistema debe implementar una lógica de adaptación contextual rural.

Esto significa que el sistema no solo debe personalizar actividades según el rendimiento académico, sino también considerar las condiciones de acceso digital del estudiante.

La decisión técnica central es que la plataforma debe ser accesible, liviana, simple y sensible al contexto rural, evitando interpretar todos los datos de bajo uso como bajo desempeño académico.

## 14. Instrucción para Cursor

Cuando desarrolles el sistema, usa este artículo para adaptar la plataforma a la realidad rural del VRAEM.

No diseñes el sistema como si todos los estudiantes tuvieran buena conexión, computadora propia y uso constante de internet.

El sistema debe permitir:

- Registrar inactividad.
- Registrar problemas de conectividad.
- Diferenciar bajo acceso de bajo rendimiento.
- Mostrar alertas al docente.
- Usar recursos livianos.
- Priorizar actividades breves.
- Evitar dependencia de materiales pesados.
- Generar reportes de participación por estudiante o grupo.

El sistema debe reflejar la idea central del artículo: la educación virtual en zonas rurales solo puede reducir desigualdades si se diseña considerando infraestructura, conectividad, formación docente, inclusión digital y condiciones reales del estudiante.
