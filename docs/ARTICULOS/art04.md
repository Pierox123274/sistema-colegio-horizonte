# ARTÍCULO 04 — FICHA TÉCNICA PARA CURSOR

## 1. Referencia

Contrino, M. F.; Reyes-Millán, M.; Vázquez-Villegas, P.; Membrillo-Hernández, J. (2024).  
“Utilizar una herramienta de aprendizaje adaptativo para mejorar el rendimiento y la satisfacción de los estudiantes en la educación en línea y presencial, con el fin de lograr un enfoque más personalizado”.  
Revista: Entornos de aprendizaje inteligentes.  
Volumen/Número: 11(6).  
Páginas: 1–24.

## 2. Idea clave del artículo

El artículo analiza la implementación de una herramienta de aprendizaje adaptativo en un curso universitario, comparando su impacto en modalidad online y presencial.

La idea principal es que el aprendizaje adaptativo puede mejorar el rendimiento académico y la satisfacción del estudiante cuando se integra de manera pedagógica en la secuencia del curso, no como una herramienta aislada.

El artículo usa la plataforma CogBooks dentro de un curso de estadística para la toma de decisiones, combinándola con CANVAS y Zoom en un modelo flexible, interactivo y apoyado por tecnología.

## 3. Aporte específico al sistema

Este artículo aporta una idea importante para el sistema: la adaptación no debe depender solo del algoritmo, sino también del docente.

El docente debe poder revisar los datos generados por la plataforma y usar esa información para ajustar su enseñanza, reforzar temas difíciles y acompañar mejor a los estudiantes.

Su aporte principal para el sistema es fortalecer la conexión entre:

- Plataforma adaptativa.
- Analíticas de aprendizaje.
- Planificación docente.
- Actividades de refuerzo.
- Rendimiento académico.
- Satisfacción del estudiante.

## 4. Decisión técnica derivada

A partir de este artículo, el sistema debe permitir que el docente use los datos del estudiante para tomar decisiones pedagógicas.

Esto significa que el sistema no debe limitarse a recomendar actividades automáticamente. También debe mostrar al docente qué estudiantes tienen dificultades, qué temas presentan mayor error y qué contenidos deben reforzarse en clase.

La decisión técnica principal es que el sistema debe incluir una capa de apoyo docente basada en analíticas de progreso y comprensión.

El artículo permite justificar el uso de datos académicos y de satisfacción para evaluar el impacto del sistema.

## 6. Técnicas o enfoques que respalda

El artículo no se centra en un algoritmo específico de machine learning, sino en la implementación pedagógica de una herramienta adaptativa.

Para el sistema, respalda enfoques como:

- Aprendizaje adaptativo.
- Analítica de aprendizaje.
- Evaluaciones automatizadas.
- Retroalimentación inmediata.
- Personalización de contenidos.
- Secuencia didáctica adaptativa.
- Uso docente de datos.
- Comparación antes/después.
- Evaluación de satisfacción estudiantil.
- Integración entre plataforma y enseñanza presencial o virtual.

Este artículo es más útil para diseñar cómo el docente usa la información del sistema que para elegir un algoritmo.

## 7. Regla o lógica aplicable al sistema

Este artículo permite derivar las siguientes reglas funcionales:

Si varios estudiantes fallan en el mismo tema, el sistema debe marcar ese tema como prioridad de refuerzo para el docente.

Si un estudiante no completa las actividades previas, el sistema debe mostrarlo como pendiente antes de avanzar.

Si el estudiante obtiene bajo desempeño en una actividad adaptativa, el sistema debe recomendar recursos alternativos o ejercicios de refuerzo.

Si el docente revisa el panel antes de una sesión, el sistema debe mostrar los temas con menor comprensión grupal.

Si el estudiante mejora después del refuerzo adaptativo, el sistema debe registrar el avance y actualizar su progreso.

Si la satisfacción del estudiante es baja, el sistema debe permitir identificar si el problema está asociado a dificultad de contenido, plataforma o acompañamiento.

## 8. Métricas útiles

El artículo evalúa el aprendizaje adaptativo mediante métricas académicas y de satisfacción.

Para el sistema, se pueden considerar:

- Puntaje en evaluación diagnóstica.
- Puntaje en evaluaciones posteriores.
- Mejora entre evaluación inicial y evaluación final.
- Calificación final.
- Porcentaje de aprobación.
- Porcentaje de estudiantes con alto rendimiento.
- Actividades completadas.
- Progreso por tema.
- Temas con mayor dificultad.
- Satisfacción del estudiante.
- Percepción de utilidad del sistema.
- Comparación entre estudiantes con y sin uso de actividades adaptativas.
- Comparación entre modalidad presencial y virtual, si aplica.

## 9. Resultado o evidencia relevante

El artículo muestra que el uso de una herramienta de aprendizaje adaptativo puede mejorar el logro académico de los estudiantes en cursos presenciales y online.

También evidencia que los estudiantes obtienen mejores resultados cuando el aprendizaje adaptativo se integra con la planificación docente y con actividades activas antes, durante y después de la clase.

Un hallazgo importante es que el aprendizaje adaptativo no solo ayuda al estudiante de forma individual, sino que también permite al docente identificar temas difíciles y rediseñar la sesión de clase usando datos del sistema.

## 10. Limitación del artículo

Aunque el artículo es útil para orientar el diseño del sistema, presenta algunas limitaciones:

- Se desarrolla en un solo curso universitario.
- Se aplica en una sola institución.
- Utiliza una herramienta específica llamada CogBooks.
- No está contextualizado en educación rural ni en la zona VRAEM.
- No propone una arquitectura de software lista para implementar.
- No desarrolla un modelo propio de machine learning.
- No se enfoca directamente en nivelación académica escolar.
- Los resultados pueden depender del diseño pedagógico del curso y del uso que haga el docente de la herramienta.
- Aunque analiza rendimiento y satisfacción, podría profundizar más en autonomía, retroalimentación y diferencias entre perfiles de estudiantes.

Por ello, el sistema debe tomar la idea de integración pedagógica y uso docente de datos, pero adaptarla al contexto de estudiantes con brechas de aprendizaje en el VRAEM.

## 11. Cómo se usará en el sistema

Este artículo se usará para justificar que la plataforma debe ayudar al docente a tomar decisiones, no solo al estudiante a resolver actividades.

Su aporte principal se aplicará en:

- Panel docente.
- Reportes de temas difíciles.
- Seguimiento del progreso.
- Identificación de estudiantes con bajo desempeño.
- Preparación de refuerzos.
- Retroalimentación inmediata.
- Evaluación de satisfacción.
- Comparación del avance antes y después de la nivelación.

No se usará principalmente para justificar un algoritmo específico, sino para justificar la integración entre sistema adaptativo y acción docente.

## 12. Aplicación directa al sistema VRAEM

En el sistema de nivelación académica adaptativa para estudiantes del VRAEM, este artículo permite definir que el docente debe tener un papel activo.

La plataforma debe:

- Mostrar qué estudiantes necesitan apoyo.
- Mostrar qué temas tienen mayor cantidad de errores.
- Permitir planificar refuerzos según datos reales.
- Ayudar a preparar actividades antes de una sesión.
- Mostrar el progreso individual y grupal.
- Permitir medir satisfacción o percepción de utilidad.
- Usar datos para mejorar la enseñanza, no solo para calificar.

Esto es importante porque en contextos vulnerables el sistema no debe reemplazar al docente, sino darle información clara para intervenir mejor.

## 13. Decisión técnica final derivada del Artículo 04

A partir de este artículo, el sistema debe implementar un panel docente con analíticas accionables.

Esto significa que los datos del estudiante deben transformarse en información útil para la enseñanza: temas que requieren refuerzo, estudiantes en riesgo, actividades pendientes, mejora de rendimiento y satisfacción del estudiante.

La decisión técnica central es que el aprendizaje adaptativo debe estar integrado al proceso pedagógico del docente.

## 14. Instrucción para Cursor

Cuando desarrolles el sistema, usa este artículo para diseñar la relación entre la plataforma y el docente.

No hagas que el sistema funcione solo como recomendador automático. El docente debe poder revisar datos y tomar decisiones.

El sistema debe permitir:

- Ver progreso individual.
- Ver progreso grupal.
- Identificar temas con baja comprensión.
- Identificar estudiantes con bajo desempeño.
- Preparar refuerzos según datos.
- Revisar actividades pendientes.
- Medir avance antes y después.
- Registrar satisfacción o percepción del estudiante.

El sistema debe reflejar la idea central del artículo: el aprendizaje adaptativo mejora cuando la tecnología se integra con la planificación docente y se usa para personalizar la enseñanza.
