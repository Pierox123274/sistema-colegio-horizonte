# ARTÍCULO 09 — FICHA TÉCNICA PARA CURSOR

## 1. Referencia

Han, J.; Liu, G.; Xiang, S. (2025).  
“Participar o no en la IA: la participación en el aprendizaje entre estudiantes de secundaria de zonas rurales en un entorno de aprendizaje adaptativo impulsado por IA”.  
Revista: Humanities and Social Sciences Communications.  
Volumen: 12.  
Artículo: 1292.

## 2. Idea clave del artículo

El artículo analiza el learning engagement de estudiantes de secundaria de zonas rurales cuando usan un sistema de aprendizaje adaptativo impulsado por inteligencia artificial.

La idea principal es que la efectividad de un sistema adaptativo con IA no depende únicamente de que el sistema personalice contenidos, sino también de que el estudiante se mantenga comprometido, motivado, perciba utilidad, pueda usar la tecnología con facilidad y no experimente una carga cognitiva excesiva.

El artículo integra tres enfoques teóricos:

- Self-Determination Theory, SDT.
- Technology Acceptance Model, TAM.
- Cognitive Load Theory, CLT.

## 3. Aporte específico al sistema

Este artículo aporta una visión centrada en estudiantes rurales, lo cual es muy importante para el sistema VRAEM.

Su aporte principal es que permite justificar que el sistema no debe evaluar únicamente rendimiento académico, sino también participación, facilidad de uso, percepción de utilidad, motivación y carga cognitiva.

Este artículo ayuda a fortalecer:

- Diseño simple y fácil de usar.
- Seguimiento del engagement.
- Medición de participación.
- Evaluación de percepción de utilidad.
- Control de sobrecarga cognitiva.
- Adaptación para estudiantes rurales.
- Detección de desmotivación o abandono.

## 4. Decisión técnica derivada

A partir de este artículo, el sistema debe incorporar indicadores de engagement y aceptación tecnológica.

Esto significa que el sistema no debe asumir que una recomendación adaptativa será efectiva solo porque está generada por IA. También debe medir si el estudiante participa, entiende la actividad, percibe utilidad y puede usar la plataforma sin dificultad.

La decisión técnica principal es que la plataforma debe adaptar no solo el contenido, sino también la complejidad, presentación y acompañamiento según el nivel de participación y facilidad de uso del estudiante.

## 7. Regla o lógica aplicable al sistema

Este artículo permite derivar las siguientes reglas funcionales:

Si el estudiante abandona actividades con frecuencia, el sistema debe registrar bajo engagement y recomendar actividades más cortas o guiadas.

Si el estudiante indica que una actividad es muy difícil, el sistema debe reducir la carga cognitiva y ofrecer explicación paso a paso.

Si el estudiante usa poco la plataforma, el sistema debe diferenciar entre baja motivación, baja facilidad de uso o problema de acceso.

Si el estudiante percibe baja utilidad, el sistema debe mostrar con mayor claridad el propósito de la actividad y cómo ayuda a mejorar.

Si el estudiante completa actividades y mejora su desempeño, el sistema debe reforzar su confianza con retroalimentación positiva.

Si varios estudiantes rurales muestran baja facilidad de uso, se debe simplificar la interfaz o reducir pasos innecesarios.

## 9. Resultado o evidencia relevante

El artículo aporta evidencia empírica en estudiantes rurales de séptimo grado que usan un sistema adaptativo con IA en matemáticas.

Su aporte más importante es que demuestra que el engagement en estos entornos depende de múltiples factores, no solo del rendimiento académico. Variables como autonomía, competencia, relación, facilidad de uso, utilidad percibida y carga cognitiva influyen en la forma en que los estudiantes participan en el sistema.

El estudio también plantea el engagement en cuatro dimensiones: conductual, cognitiva, emocional y social. Esto permite analizar la participación de forma más completa que solo contando accesos o notas.

## 10. Limitación del artículo

Aunque el artículo es muy útil para el sistema VRAEM, presenta algunas limitaciones:

- Está contextualizado en zonas rurales de China, no en el VRAEM.
- Se centra en estudiantes de séptimo grado y matemáticas.
- Usa cuestionarios para medir percepciones, lo cual depende de respuestas sinceras.
- No propone una arquitectura de software lista para implementar.
- No entrega un modelo machine learning entrenado para nivelación académica.
- No aborda directamente comprensión lectora u otras áreas.
- No profundiza en conectividad limitada como problema técnico central.
- Su aporte es más explicativo y pedagógico que algorítmico.

Por ello, el sistema debe tomar sus factores de engagement como variables de diseño y seguimiento, pero adaptarlos al contexto local.

## 12. Aplicación directa al sistema VRAEM

En el sistema de nivelación académica adaptativa para estudiantes del VRAEM, este artículo permite definir que el estudiante debe sentirse capaz de usar la plataforma y comprender su utilidad.

La plataforma debe:

- Ser fácil de usar.
- Presentar instrucciones claras.
- Evitar actividades demasiado extensas.
- Reducir la carga cognitiva cuando el estudiante falla.
- Mostrar retroalimentación positiva.
- Registrar abandono o baja participación.
- Medir si el estudiante percibe utilidad.
- Alertar al docente cuando haya bajo engagement.
- Diferenciar bajo rendimiento de baja motivación o dificultad de uso.

Esto es importante porque en contextos rurales una plataforma con IA puede fracasar si el estudiante no la entiende, no la considera útil o se siente sobrecargado.

## 13. Decisión técnica final derivada del Artículo 09

A partir de este artículo, el sistema debe implementar una lógica de adaptación sensible al engagement.

Esto significa que las recomendaciones no deben depender solo del puntaje académico, sino también de señales de participación, facilidad de uso, motivación y carga cognitiva.

La decisión técnica central es que el sistema debe monitorear cuatro dimensiones:

- Participación conductual.
- Participación cognitiva.
- Participación emocional.
- Participación social.

Estas dimensiones permitirán tomar mejores decisiones sobre cuándo reforzar, simplificar, motivar o alertar al docente.

## 14. Instrucción para Cursor

Cuando desarrolles el sistema, usa este artículo para implementar indicadores de engagement y aceptación tecnológica.

No hagas que el sistema solo analice calificaciones. También debe observar:

- Si el estudiante ingresa.
- Si completa actividades.
- Si abandona actividades.
- Si percibe dificultad.
- Si entiende la utilidad de la actividad.
- Si necesita apoyo.
- Si muestra baja continuidad.

El sistema debe reflejar la idea central del artículo: en estudiantes rurales, un sistema adaptativo con IA solo será efectivo si además de personalizar contenidos logra mantener engagement, facilidad de uso, motivación y baja carga cognitiva.
