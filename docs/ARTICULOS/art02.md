# ARTÍCULO 02 — FICHA TÉCNICA PARA CURSOR

## 1. Referencia

Jing, Y.; Zhao, L.; Zhu, K.; Wang, H.; Wang, C.; Xia, Q. (2023).  
“Panorama de la investigación sobre el aprendizaje adaptativo en la educación: un estudio bibliométrico de las publicaciones de investigación desde el año 2000 hasta el 2022”.  
Revista: Sustainability.  
Volumen/Número: 15(4).  
Año: 2023.

## 2. Idea clave del artículo

El artículo analiza la evolución del aprendizaje adaptativo en educación entre los años 2000 y 2022 mediante un estudio bibliométrico. Su idea principal es que el aprendizaje adaptativo se ha consolidado como un campo interdisciplinario impulsado por tecnologías como inteligencia artificial, big data, deep learning, sistemas tutoriales inteligentes y modelos computacionales.

El artículo muestra que el aprendizaje adaptativo ha pasado de enfoques básicos de evaluación y enseñanza automatizada hacia sistemas inteligentes capaces de personalizar el proceso educativo según las necesidades, capacidades y características del estudiante.

## 3. Aporte específico al sistema

Este artículo aporta una visión panorámica sobre las principales tendencias tecnológicas del aprendizaje adaptativo. Sirve para justificar que el sistema de nivelación académica debe apoyarse en técnicas actuales de inteligencia artificial y análisis de datos, no solo en reglas estáticas.

Su aporte principal para el sistema es ayudar a identificar qué enfoques tecnológicos son relevantes para construir una plataforma adaptativa, especialmente:

- Modelos de aprendizaje adaptativo basados en inteligencia artificial.
- Uso de deep learning y análisis de datos educativos.
- Sistemas de tutoría inteligente.
- Modelado y seguimiento del aprendizaje.
- Análisis de tendencias para orientar futuras mejoras del sistema.

## 4. Decisión técnica derivada

A partir de este artículo, el sistema debe diseñarse como una solución evolutiva y escalable. Primero puede iniciar con reglas simples y modelos interpretables, pero debe estar preparado para incorporar progresivamente técnicas más avanzadas como análisis de datos educativos, clustering, modelos predictivos, sistemas de recomendación o deep learning cuando exista suficiente información acumulada.

La decisión técnica principal es que el sistema no debe limitarse a registrar notas, sino que debe generar información analítica sobre el aprendizaje del estudiante para orientar rutas de nivelación más precisas.

## 5. Datos que sugiere considerar

El artículo no propone una base de datos específica para estudiantes, pero por su enfoque en aprendizaje adaptativo y análisis de datos educativos, permite justificar el uso de variables como:

- Nivel académico del estudiante.
- Puntajes obtenidos en evaluaciones.
- Historial de desempeño.
- Progreso por tema.
- Temas dominados.
- Temas con dificultad.
- Frecuencia de participación.
- Interacciones con la plataforma.
- Evolución del aprendizaje en el tiempo.
- Resultados antes y después de la nivelación.
- Patrones de comportamiento académico.

## 6. Técnicas que respalda

El artículo identifica tendencias y líneas tecnológicas asociadas al aprendizaje adaptativo. Para el sistema, respalda el uso de:

- Inteligencia artificial.
- Machine learning.
- Deep learning.
- Big data educativo.
- Sistemas tutoriales inteligentes.
- Modelado computacional.
- Feature extraction.
- Modelos adaptativos.
- Análisis de co-ocurrencia o patrones.
- Analítica de aprendizaje.

Para el MVP del sistema, estas técnicas pueden traducirse inicialmente en modelos simples de clasificación o recomendación. En fases posteriores, pueden evolucionar hacia modelos más avanzados de predicción y personalización.

## 7. Regla o lógica aplicable al sistema

El artículo permite derivar la siguiente lógica para el sistema:

Si el sistema acumula datos históricos del estudiante, debe analizarlos para identificar patrones de aprendizaje y ajustar las recomendaciones de manera progresiva.

Si varios estudiantes presentan dificultades en un mismo tema, el sistema debe registrar ese tema como una brecha frecuente y mostrarlo al docente.

Si el desempeño del estudiante mejora con determinadas actividades, el sistema debe priorizar actividades similares o de mayor dificultad.

Si el estudiante mantiene bajo rendimiento en varias sesiones, el sistema debe marcarlo como posible caso de riesgo académico.

Si el sistema detecta patrones repetidos en el grupo, debe generar información útil para la toma de decisiones del docente.

## 8. Métricas útiles

El artículo trabaja principalmente con métricas bibliométricas, pero sus hallazgos permiten orientar métricas para evaluar el sistema. Las métricas útiles son:

- Progreso académico del estudiante.
- Mejora del puntaje entre evaluación inicial y evaluación posterior.
- Reducción de brechas de aprendizaje.
- Frecuencia de uso del sistema.
- Participación del estudiante.
- Finalización de actividades recomendadas.
- Evolución del rendimiento por tema.
- Identificación de patrones grupales.
- Cantidad de estudiantes que mejoran de nivel.
- Precisión de las recomendaciones generadas por el sistema.

## 9. Limitación del artículo

El artículo es útil como mapa general del campo, pero tiene limitaciones para el desarrollo directo del sistema:

- Es un estudio bibliométrico, no una implementación de software.
- No propone una arquitectura técnica concreta.
- No presenta un modelo de machine learning listo para usar.
- No evalúa directamente estudiantes en una plataforma real.
- Se centra en publicaciones científicas, autores, países y tendencias.
- No aborda específicamente contextos vulnerables como el VRAEM.
- No profundiza en conectividad limitada, ruralidad o desigualdad educativa.
- Tiene un enfoque más tecnológico y cuantitativo que pedagógico aplicado.

Por ello, debe usarse como respaldo para orientar tendencias y decisiones generales, pero no como guía única de implementación.

## 10. Cómo se usará en el sistema

Este artículo se usará para justificar que el sistema debe alinearse con las tendencias actuales del aprendizaje adaptativo: inteligencia artificial, análisis de datos educativos, modelado del aprendizaje y sistemas inteligentes de personalización.

Su aporte principal se aplicará en la planificación evolutiva del sistema. En el MVP, servirá para justificar el uso de reglas, clasificación y reportes analíticos. En versiones posteriores, servirá como fundamento para incorporar modelos más avanzados de machine learning, análisis de patrones, predicción de riesgo académico y recomendaciones más precisas.

El artículo también permite sustentar que el sistema debe generar reportes no solo individuales, sino también grupales, porque el análisis de patrones permite identificar tendencias de aprendizaje y brechas comunes en los estudiantes.
