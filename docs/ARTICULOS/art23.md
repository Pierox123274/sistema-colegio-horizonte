# ARTÍCULO 23 — FICHA TÉCNICA PARA CURSOR

## 1. Referencia

Tzimas, D. E.; Demetriadis, S. N. (2024).  
“Impact of Learning Analytics Guidance on Student Self-Regulated Learning Skills, Performance, and Satisfaction: A Mixed Methods Study”.  
Revista: Education Sciences.  
Volumen/Número: 14(1).  
Páginas: 1–18.

## 2. Idea clave del artículo

El artículo analiza el impacto de la analítica del aprendizaje —Learning Analytics— cuando se combina con distintos niveles de guía docente. Su idea principal es que mostrar datos al estudiante no es suficiente; esos datos deben ir acompañados de orientación, retroalimentación e intervención pedagógica para mejorar el aprendizaje autorregulado, el rendimiento académico y la satisfacción del estudiante.

El artículo compara dos enfoques:

- Guía mínima: el estudiante recibe datos básicos como notas, estadísticas o información general de desempeño.
- Guía fuerte: el estudiante recibe indicadores personalizados, retroalimentación, orientación docente y entrevistas o acompañamiento para reflexionar sobre su aprendizaje.

## 3. Aporte específico al sistema

Este artículo aporta la idea de que el sistema no debe limitarse a mostrar resultados o reportes automáticos. También debe ayudar al estudiante a interpretar su desempeño y tomar decisiones para mejorar.

Su aporte principal es fortalecer la parte de:

- Retroalimentación personalizada.
- Autorregulación del aprendizaje.
- Orientación del estudiante.
- Intervención docente basada en datos.
- Uso pedagógico de la analítica del aprendizaje.

## 4. Decisión técnica derivada

A partir de este artículo, el sistema debe incorporar una capa de orientación pedagógica sobre los datos.

Esto significa que, cuando el sistema detecte una brecha de aprendizaje, bajo rendimiento o poco avance, no debe mostrar solo un número o una nota. Debe generar una recomendación clara y comprensible para el estudiante y también una alerta útil para el docente.

La decisión técnica principal es que los reportes y dashboards deben ser accionables, no solo informativos.

## 5. Datos que sugiere considerar

El artículo permite justificar el uso de datos relacionados con desempeño, participación y autorregulación. Para el sistema, se pueden considerar:

- Calificaciones o puntajes.
- Progreso por tema.
- Actividades completadas.
- Actividades pendientes.
- Frecuencia de ingreso.
- Tiempo de dedicación.
- Participación en actividades.
- Historial de evaluaciones.
- Temas con bajo desempeño.
- Comparación del avance esperado frente al avance real.
- Nivel de satisfacción del estudiante.
- Percepción de utilidad de la retroalimentación.
- Necesidad de apoyo docente.

## 6. Técnicas que respalda

El artículo no se centra en un algoritmo específico de machine learning, sino en el uso pedagógico de Learning Analytics.

Para el sistema, respalda el uso de:

- Learning Analytics.
- Dashboards educativos.
- Indicadores tipo semáforo.
- Retroalimentación personalizada.
- Monitoreo del desempeño.
- Alertas tempranas.
- Seguimiento docente.
- Evaluación de autorregulación.
- Métodos mixtos para evaluar impacto.

Este artículo es más útil para diseñar la capa de seguimiento y retroalimentación que para elegir un algoritmo de predicción.

## 7. Regla o lógica aplicable al sistema

Este artículo permite derivar las siguientes reglas funcionales:

Si el estudiante obtiene bajo rendimiento, el sistema debe mostrar una recomendación clara sobre qué tema debe reforzar.

Si el estudiante tiene actividades pendientes, el sistema debe mostrar una alerta y sugerir una acción concreta.

Si el estudiante no mejora después de varias actividades, el sistema debe notificar al docente para una intervención más directa.

Si el estudiante presenta bajo progreso, el sistema debe mostrar su estado mediante indicadores simples, por ejemplo: verde, amarillo o rojo.

Si el estudiante recibe retroalimentación, esta debe ser positiva, orientadora y enfocada en acciones concretas de mejora.

Si el docente revisa el panel, debe poder identificar rápidamente quién necesita apoyo y por qué.

## 8. Métricas útiles

El artículo trabaja con variables relacionadas con rendimiento, autorregulación y satisfacción. Para el sistema, se pueden considerar estas métricas:

- Mejora del rendimiento académico.
- Nivel de autorregulación del estudiante.
- Cumplimiento de actividades.
- Participación en la plataforma.
- Progreso por tema.
- Satisfacción del estudiante con la retroalimentación.
- Percepción de utilidad del sistema.
- Cantidad de alertas generadas.
- Cantidad de intervenciones docentes realizadas.
- Reducción de estudiantes en riesgo.
- Comparación entre progreso antes y después de recibir retroalimentación.

## 9. Resultado o evidencia relevante

El artículo muestra que Learning Analytics puede ser más efectivo cuando se combina con una guía docente fuerte. La analítica por sí sola ayuda a visualizar datos, pero no garantiza que el estudiante sepa interpretarlos o convertirlos en acciones de mejora.

También evidencia que el acompañamiento docente, la retroalimentación personalizada y la orientación explícita pueden favorecer la autorregulación, la satisfacción y el rendimiento académico.

Sin embargo, el estudio también advierte que el uso de analítica puede generar efectos emocionales negativos en algunos estudiantes, como ansiedad, estrés o incomodidad cuando se comparan con otros.

## 10. Limitación del artículo

Aunque el artículo es útil para orientar el diseño del sistema, presenta algunas limitaciones:

- Fue aplicado en un solo curso universitario.
- La muestra fue de 93 estudiantes, por lo que los resultados deben interpretarse con cautela.
- El contexto fue educación superior en línea, no educación escolar rural.
- El LMS solo registró una parte de las actividades de aprendizaje.
- No considera completamente factores sociales, familiares o económicos.
- No está contextualizado en la zona VRAEM.
- El uso de datos puede generar presión o ansiedad si se presenta de forma inadecuada.

Por ello, el sistema debe usar la analítica del aprendizaje de forma cuidadosa, evitando comparaciones negativas entre estudiantes y priorizando mensajes de apoyo.

## 11. Cómo se usará en el sistema

Este artículo se usará para justificar que el sistema debe tener reportes comprensibles, retroalimentación personalizada y apoyo docente basado en datos.

No se usará principalmente para justificar algoritmos de machine learning, sino para mejorar la forma en que el sistema comunica los resultados al estudiante y al docente.

Su aporte principal se aplicará en:

- Diseño de dashboards.
- Indicadores de progreso.
- Alertas académicas.
- Retroalimentación automática.
- Seguimiento docente.
- Apoyo a la autorregulación del estudiante.

## 12. Aplicación directa al sistema VRAEM

En el sistema de nivelación académica para estudiantes del VRAEM, este artículo permite definir que los datos deben usarse para orientar, no para presionar.

La plataforma debe:

- Mostrar al estudiante qué está mejorando.
- Mostrar qué temas necesita reforzar.
- Recomendar acciones concretas.
- Evitar comparaciones públicas entre estudiantes.
- Usar mensajes motivadores.
- Alertar al docente cuando un estudiante necesite ayuda.
- Permitir que el docente complemente la recomendación automática con orientación pedagógica.

Esto es importante porque los estudiantes con brechas de aprendizaje no solo necesitan actividades adaptativas, sino también acompañamiento y retroalimentación clara.

## 13. Decisión técnica final derivada del Artículo 23

A partir de este artículo, el sistema debe implementar una analítica del aprendizaje orientada a la acción pedagógica.

Esto significa que los datos del estudiante deben transformarse en recomendaciones comprensibles, alertas útiles y retroalimentación personalizada.

La decisión técnica central es que el sistema no debe mostrar únicamente estadísticas; debe ayudar al estudiante y al docente a tomar decisiones para mejorar el aprendizaje.

## 14. Instrucción para Cursor

Cuando desarrolles el sistema, usa este artículo para diseñar la capa de Learning Analytics y retroalimentación.

No hagas dashboards solo con gráficos o números. Cada indicador debe responder:

- Qué está pasando con el estudiante.
- Qué tema necesita reforzar.
- Qué acción debe realizar.
- Cuándo debe intervenir el docente.
- Qué recomendación debe recibir el estudiante.

El sistema debe reflejar la idea central del artículo: la analítica del aprendizaje funciona mejor cuando los datos se convierten en guía, retroalimentación y acompañamiento pedagógico.
