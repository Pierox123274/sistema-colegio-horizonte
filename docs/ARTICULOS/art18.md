# ARTÍCULO 18 — FICHA TÉCNICA PARA CURSOR

## 1. Referencia

Son, T. (2024).  
“Sistemas de tutoría inteligente en la enseñanza de las matemáticas: una revisión sistemática de la literatura utilizando el modelo SAMR”.  
Revista: Computers.  
Volumen/Número: 13(10).  
Artículo: 102474.

## 2. Idea clave del artículo

El artículo analiza el uso de Sistemas Tutores Inteligentes —ITS— en la enseñanza de las matemáticas mediante una revisión sistemática basada en el modelo SAMR.

La idea principal es que los ITS pueden mejorar el aprendizaje matemático mediante personalización, retroalimentación inmediata, apoyo paso a paso y adaptación del contenido. Sin embargo, el artículo advierte que muchas implementaciones no transforman realmente la enseñanza, sino que solo digitalizan o refuerzan prácticas tradicionales.

El modelo SAMR se usa para clasificar el nivel de integración tecnológica:

- Sustitución.
- Aumento.
- Modificación.
- Redefinición.

## 3. Aporte específico al sistema

Este artículo aporta una perspectiva pedagógica crítica para el sistema. Su valor principal es que permite evitar que la plataforma sea solo una versión digital de ejercicios tradicionales.

Para el sistema VRAEM, el artículo ayuda a justificar que la tecnología debe usarse para transformar la nivelación académica, no solo para mostrar preguntas en pantalla.

El aporte se centra en:

- Tutoría inteligente en matemáticas.
- Retroalimentación inmediata.
- Apoyo paso a paso.
- Integración pedagógica del sistema.
- Rol activo del docente.
- Diseño de actividades que superen la simple repetición.
- Evaluación del nivel de innovación tecnológica mediante SAMR.

## 4. Decisión técnica derivada

A partir de este artículo, el sistema debe diseñar actividades matemáticas que no solo sustituyan fichas impresas o ejercicios tradicionales.

La decisión técnica principal es que las actividades deben buscar al menos el nivel de “aumento” y, cuando sea posible, “modificación” dentro del modelo SAMR.

Esto significa que el sistema debe:

- No limitarse a mostrar ejercicios.
- Dar retroalimentación inmediata.
- Ofrecer pistas.
- Adaptar la dificultad.
- Registrar errores frecuentes.
- Permitir seguimiento docente.
- Rediseñar la forma de reforzar matemáticas según los datos del estudiante.

## 7. Regla o lógica aplicable al sistema

Este artículo permite derivar las siguientes reglas funcionales:

Si el estudiante responde mal un ejercicio matemático, el sistema debe ofrecer retroalimentación inmediata y no esperar hasta el final de la evaluación.

Si el estudiante falla varias veces en el mismo tipo de operación, el sistema debe registrar el patrón de error y recomendar una actividad de refuerzo.

Si el estudiante solicita muchas pistas, el sistema debe marcar ese tema como dificultad persistente.

Si varios estudiantes fallan en el mismo contenido, el sistema debe mostrar al docente que ese tema requiere intervención pedagógica.

Si una actividad solo muestra preguntas y respuestas, se considera nivel de sustitución. Para mejorarla, debe agregar retroalimentación, pistas, adaptación o seguimiento.

Si una actividad adapta la ruta y permite al docente rediseñar el refuerzo, se acerca a niveles más altos del modelo SAMR.

## 9. Resultado o evidencia relevante

El artículo revisa estudios empíricos sobre ITS en matemáticas entre 2003 y 2023.

El resultado principal es que la mayoría de estudios se ubican en el nivel de aumento del modelo SAMR, con 67.3%. Solo 1.7% corresponde a sustitución, 24.1% a modificación y 6.9% a redefinición.

Esto significa que la mayoría de ITS mejora prácticas existentes, pero pocos transforman profundamente la enseñanza matemática.

También se identifica que los estudios se concentran principalmente en primaria y secundaria, especialmente en números, aritmética, álgebra y geometría. Además, el rol docente suele ser limitado, lo cual reduce el potencial transformador de estas herramientas.

## 10. Limitación del artículo

Aunque el artículo es útil para orientar el diseño pedagógico del sistema, presenta algunas limitaciones:

- Es una revisión sistemática, no una implementación directa de software.
- No propone una arquitectura técnica lista para desarrollar.
- No entrega un modelo machine learning entrenado.
- Se enfoca en matemáticas y no en todas las áreas académicas.
- Usa principalmente artículos en inglés y revistas revisadas por pares.
- Depende del modelo SAMR, que puede simplificar procesos pedagógicos complejos.
- No está contextualizado en la zona VRAEM.
- No desarrolla actividades concretas de nivelación para estudiantes con rezago.
- No profundiza en conectividad limitada o condiciones rurales.

Por ello, el sistema debe usar este artículo como guía para mejorar el diseño pedagógico de las actividades matemáticas, pero adaptándolo a una solución viable para el VRAEM.

## 11. Cómo se usará en el sistema

Este artículo se usará para justificar que el sistema debe incorporar una tutoría matemática más inteligente y pedagógicamente integrada.

No se usará principalmente para elegir un algoritmo, sino para orientar cómo deben diseñarse las actividades matemáticas dentro del sistema.

Su aporte principal se aplicará en:

- Diseño de ejercicios matemáticos adaptativos.
- Retroalimentación inmediata.
- Pistas por error.
- Registro de patrones de dificultad.
- Seguimiento del docente.
- Evaluación del nivel de integración tecnológica.
- Mejora de actividades para que no sean solo ejercicios digitalizados.

## 12. Aplicación directa al sistema VRAEM

En el sistema de nivelación académica adaptativa para estudiantes del VRAEM, este artículo permite definir que las actividades matemáticas deben ayudar al estudiante durante el proceso, no solo calificarlo al final.

La plataforma debe:

- Presentar ejercicios según el nivel del estudiante.
- Dar retroalimentación inmediata.
- Mostrar pistas cuando exista error.
- Registrar tipos de errores frecuentes.
- Recomendar ejercicios previos si el estudiante no domina una base.
- Permitir al docente ver qué temas matemáticos requieren refuerzo.
- Diseñar actividades que promuevan comprensión, no solo repetición.

Esto es importante porque muchos estudiantes con brechas matemáticas necesitan acompañamiento paso a paso y no únicamente más ejercicios.

## 13. Decisión técnica final derivada del Artículo 18

A partir de este artículo, el sistema debe implementar actividades matemáticas con lógica de Sistema Tutor Inteligente.

Esto significa que las actividades deben incluir retroalimentación inmediata, pistas, registro de errores y adaptación de dificultad.

La decisión técnica central es que el sistema debe evitar quedarse en el nivel de “sustitución” tecnológica. Debe avanzar hacia una integración más significativa, donde la tecnología mejore y rediseñe la forma de nivelar matemáticas.

## 14. Instrucción para Cursor

Cuando desarrolles el sistema, usa este artículo para diseñar actividades matemáticas adaptativas con enfoque de tutoría inteligente.

No hagas que las actividades sean solo preguntas con alternativas. Cada actividad matemática debe poder incluir:

- Nivel de dificultad.
- Tipo de error.
- Retroalimentación inmediata.
- Pistas.
- Intentos.
- Explicación breve.
- Registro de avance.
- Recomendación de refuerzo.

El sistema debe reflejar la idea central del artículo: los sistemas tutores inteligentes en matemáticas solo aportan verdadero valor cuando se integran pedagógicamente y no se limitan a digitalizar prácticas tradicionales.
