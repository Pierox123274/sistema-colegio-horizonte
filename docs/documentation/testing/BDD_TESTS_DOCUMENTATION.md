# Pruebas BDD (Gherkin)

## Ubicación

`tests/Bdd/features/` — 24+ archivos `.feature`.

## Ejemplos de features

- `authentication.feature`  
- `students.feature`, `enrollments.feature`, `payments.feature`  
- `virtual_classroom.feature`, `adaptive_learning.feature`  
- `ai_tutor.feature`, `advanced_ai.feature`  
- `cms_management.feature`  
- `security_audit.feature`  
- `notifications.feature`, `virtual_meetings.feature`  
- `external_integrations.feature`  
- `platform_quality_assurance.feature`

## Propósito

Expresar comportamiento en **lenguaje natural** (español) para validación con docentes o jurado no técnico.

## Ejecución

Según configuración del proyecto (Behat o integración PHPUnit con tags). Consultar `docs/TESTING_STRATEGY.md` para el runner activo.

## Importancia

Puente entre requisitos institucionales y automatización.
